## Agent Teams TMUX

### Как сделать так, чтобы `claude` автоматически запускался в tmux

**1.** Открой файл `~/.zshrc` в любом редакторе:

```bash
nano ~/.zshrc
```

**2.** Добавь в конец файла этот код:

```bash
# Auto-launch claude inside tmux
claude() {
  if [ -z "$TMUX" ]; then
    tmux new-session -A -s main command claude "$@"
  else
    command claude "$@"
  fi
}
```

**3.** Сохрани файл (`Ctrl+O`, затем `Enter`, затем `Ctrl+X`)

**4.** Примени изменения в текущем терминале:

```bash
source ~/.zshrc
```

---

**Как это работает:**

- `[ -z "$TMUX" ]` — проверяет, находишься ли ты уже внутри tmux
- Если нет — создаёт/присоединяет tmux-сессию с именем `main` и запускает claude внутри
- Если да — запускает claude напрямую, без лишней вложенности

---

> **Коротко о главном:** Представь, что вместо одного сотрудника, которому ты даёшь задание, у тебя работает целая команда специалистов — и они сами между собой общаются, делегируют задачи и выдают готовый результат. Именно это и делает Claude Agent Teams.

## Что такое Agent Teams

**Agent Teams** — это новая функция внутри **Claude Code**, которая позволяет запускать не одного AI-ассистента, а целую команду автономных агентов, каждый из которых выполняет свою роль.

Представь обычную рабочую команду в компании:

- Есть **исследователь** — он ищет факты и данные
- Есть **копирайтер** — он пишет тексты
- Есть **дизайнер** — он создаёт промпты для изображений
- Есть **ревьюер** — он проверяет ошибки
- Есть **руководитель** — он координирует всех

Так вот, Agent Teams — это именно такая команда, только из AI-агентов. Ты даёшь одно задание, а дальше они **сами между собой договариваются**, обмениваются информацией и выдают тебе готовый результат.

> **Ключевая идея:** Раньше ты просил одну нейросеть быть одновременно и стратегом, и копирайтером, и аналитиком. Это как просить одного человека быть сразу врачом, поваром и инженером. Теперь у каждого есть своя специализация.

---

## Чем это отличается от обычного чата с AI

### Раньше (обычный промпт или субагенты):

| Проблема | Как это выглядело |
| --- | --- |
| Один промпт — один контекст | Нейросеть "забывала" часть задачи, путала контекст |
| Субагенты не общались между собой | Нужно было самому передавать данные от одного к другому |
| Ты контролировал каждый шаг | Огромное количество диалогов и ручной работы |
| Агенты не могли работать параллельно | Всё шло по очереди, долго |

### Сейчас (Agent Teams):

| Возможность | Что это даёт |
| --- | --- |
| Агенты общаются напрямую друг с другом | Копирайтер сам пишет ресёрчеру, если ему не хватает данных |
| Работа идёт параллельно | Все агенты работают одновременно, не ждут друг друга |
| Автономность | Ты просто даёшь задачу — они сами всё организуют |
| Есть руководитель (TeamLead) | Один главный агент координирует всю команду |

> **Пример:** Копирайтер пишет текст → понимает, что ему не хватает данных → сам пишет ресёрчеру → ресёрчер добавляет факты → ревьюер проверяет → возвращает на доработку. И всё это **без твоего участия**.

## Что нужно для запуска

### 1. Подписка на Claude

- **Claude Pro** — подойдёт для простых задач на модели Sonnet (~2–4 задачи в день в режиме Agent Teams)
- **Claude Max** — рекомендуется для частого использования или сложных задач на модели Opus

### 2. VS Code

Бесплатный редактор кода от Microsoft. Именно в нём работает Claude Code.

- Скачать: https://code.visualstudio.com

### 3. Claude Code

Расширение, которое устанавливается внутри VS Code и даёт доступ к Claude прямо в терминале.

## Пошаговая настройка

### Шаг 1 — Создай конфигурационный файл

По умолчанию функция Agent Teams **выключена**. Чтобы активировать:

1. Создай папку `.claude` в своём проекте
2. Внутри создай файл `config.json` с нужными настройками

### Шаг 2 — Установи Tmux

**Tmux** разделяет терминал на несколько окон одновременно. Каждый агент работает в своём окне параллельно.

```bash
brew install tmux
```

Запуск:

```bash
tmux
```

После запуска внизу экрана появится зелёная полоска — ты внутри Tmux. ✅

### Шаг 3 — Настрой автозапуск Tmux (опционально)

Чтобы Tmux запускался автоматически при каждом открытии терминала — добавь блок кода в конфигурационный файл оболочки.

### Шаг 4 — Подготовь промпт для агентов

Создай файл `prompt.md` в проекте с инструкцией для Claude.

**Пример простого промпта:**

```
Запусти трёх агентов параллельно внутри Tmux:
1. Ресёрчер — находит 5 ключевых фактов о B2B продажах в 2025
2. Копирайтер — пишет email для рассылки
3. Дизайнер — создаёт промпты для Midjourney
```

## Запуск первой команды агентов

1. Открой терминал в VS Code
2. Убедись, что работаешь внутри Tmux (видна зелёная строка)
3. Скопируй промпт из файла `prompt.md`
4. Вставь в чат Claude Code
5. Нажми Enter

**Что происходит дальше:**

- Claude создаёт отдельные окна для каждого агента
- В основном окне — главный агент (TeamLead)
- В дополнительных — субагенты
- Все начинают работать одновременно

### Переключение между окнами:

| Действие | Комбинация клавиш |
| --- | --- |
| Переключиться между окнами | `Ctrl + B` → стрелки |
| Перейти вниз | `Ctrl + B` → ↓ |
| Перейти вверх | `Ctrl + B` → ↑ |

> Ты можешь зайти в окно любого агента и написать ему дополнительную инструкцию прямо во время работы.

### Режимы подтверждений (Shift + Tab):

- **Don't ask** — агенты работают сами, но без доступа к файловой системе
- **Accept edits** — агенты могут создавать и редактировать файлы

## Разбор результата (простой пример)

После завершения все агенты закрывают окна, Claude выдаёт итоговый отчёт.

**Пример: три файла от трёх агентов**

- `draft.txt` — email для рассылки (копирайтер)
- `facts.txt` — 5 фактов о B2B продажах (ресёрчер)
- `image.txt` — промпты для Midjourney (дизайнер)

## Продвинутый пример — 6 агентов и PDF-отчёт

### Состав команды:

| Агент | Роль |
| --- | --- |
| Шоураннер | Главный координатор всей команды |
| Веб-ресёрчер | Ищет информацию в интернете |
| Факт-чекер | Проверяет достоверность найденных фактов |
| Скрипт-архитект | Составляет структуру и сценарий |
| Визуал-директор | Отвечает за визуальную часть |
| PDF-продюсер | Собирает всё в финальный PDF-документ |

### Структура файлов проекта:

```
.claude/
  └── agents/
       ├── showrunner.md
       ├── web-researcher.md
       ├── fact-checker.md
       ├── script-architect.md
       ├── visual-director.md
       └── pdf-producer.md
CLAUDE.md  ← главный файл с инструкциями и оркестрацией
```

### Flow зависимостей между агентами:

```
Ты пишешь: "Найди информацию и сделай PDF-отчёт"
        ↓
Шоураннер запускает веб-ресёрчера и факт-чекера параллельно
        ↓
Веб-ресёрчер ищет факты → передаёт факт-чекеру
        ↓
Факт-чекер проверяет результат
        ↓
Скрипт-архитект ждёт verified claims → получает → пишет структуру
        ↓
PDF-продюсер получает все материалы → собирает финальный документ
        ↓
Готовый PDF в папке outputs/
```

### Файлы в `agent-runtime/`:

- `research_raw` — сырые данные от ресёрчера
- `verified_claims` — проверенные факты
- `fact_check_notes` — заметки факт-чекера
- `brief_template` — структура проекта
- `script` — готовый сценарий

## Токены и подписки

| Подписка | Что можно делать |
| --- | --- |
| **Claude Pro** | ~2–4 задачи в режиме Agent Teams на Sonnet 4.5 за день |
| **Claude Max** | Рекомендуется для частого использования или модели Opus |

**Пример:** 4–5 задач подряд на Sonnet 4.5 с подпиской Pro = **64% дневного лимита токенов**.

> **Важно:** Agent Teams потребляет значительно больше токенов, чем обычный чат. Используй осознанно — только для задач, которые того стоят.

## Когда использовать, а когда нет

### Agent Teams подходит для:

- Сложных многоуровневых проектов
- Задач, где нужна кросс-валидация (проверка одного агента другим)
- Создания комплексных материалов: исследование + текст + дизайн
- Проектов, где важна автономность и параллельная работа
- Задач, которые раньше занимали несколько дней

### Agent Teams НЕ нужен для:

- Исправления двух строк кода
- Простого вопроса к AI
- Небольших локальных задач
- Ситуаций, когда лимит токенов критичен

> **Аналогия:** Запускать Agent Teams для мелкой задачи — всё равно что нанимать директора компании, чтобы поменять лампочку в квартире.

## Итог и главное правило

**Agent Teams — это:**

- Один из самых мощных инструментов в AI на сегодняшний день
- Команда агентов, которые общаются между собой без твоего участия
- То, что раньше занимало дни контроля — теперь делается за 10 минут
- Реальная замена нескольким специалистам для конкретных задач

**Главное правило:**

> Всегда проверяй результат сам. Агенты умные, но всё ещё могут ошибаться.
