# Message Template

Используй этот формат для всех handoff-сообщений.
Имя файла: `from-<агент>-to-<агент>.md`

---

```markdown
---
id:        <уникальный идентификатор, например from-lead-to-system-designer-001>
from:      <имя агента-отправителя>
to:        <имя агента-получателя>
type:      assignment | handoff | revision_request | approval | rejection | blocker
topic:     <одна строка — суть сообщения>
artifacts:
  - <путь к артефакту 1>
  - <путь к артефакту 2>
needs:     <что именно нужно сделать получателю>
deadline:  <следующий этап или условие завершения>
notes:     <дополнительный контекст, допущения, блокеры>
---

## Контекст

<Краткое описание ситуации: почему это сообщение отправляется сейчас>

## Что сделано

<Список выполненных действий или принятых решений>

## Что требуется

<Конкретные действия от получателя>
```

---

## Типы сообщений

| type | Когда использовать |
|---|---|
| `assignment` | lead назначает задачу агенту |
| `handoff` | агент передаёт завершённую работу следующему |
| `revision_request` | qa-engineer или bug-hunter возвращает работу на доработку |
| `approval` | qa-engineer или lead подтверждает завершение этапа |
| `rejection` | критическая проблема, требующая переделки |
| `blocker` | агент не может продолжить без внешнего решения |
